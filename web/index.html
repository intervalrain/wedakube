01 /
An ops console,
not a cheat-sheet.
Everything in the SOP — inspect, restart, scale, roll back, swagger — wrapped in one terminal UI so you never memorise another kubectl incantation.
⌗

One console, every host

A k9s-style list over SSH ControlMaster — one reused connection keeps it instant. Hosts carry their own key, user & namespace; nothing lives on your laptop.

⇡

One key, full pipeline

buildx linux/amd64 → push harbor → set image → a live rollout bar driven by real replica counts → auto-rollback with captured logs when it fails.

⊘

Safe by default

Read-only until you flip WRITE mode; every change shows the exact kubectl command first. Protected namespaces are locked. No cluster delete.

◈

Swagger in a keystroke

Press w and WedaKube opens a port-forward through an SSH tunnel straight to your local browser — then tears it down on exit. Zero drift, zero NodePort.

From dotnet build to running pod.
The same five steps you'd run by hand — minus the three gotchas that bite every compose user moving to k3s.
STEP 01

Tag

v0.1.0_20260602.1 — immutable per build, because pullPolicy is IfNotPresent.

▶
STEP 02

Build

buildx --platform linux/amd64 — your arm64 Mac, their amd64 node.

▶
STEP 03

Push

Straight to harbor in the same step — no stale local cache.

▶
STEP 04

Set image

Over SSH on the node — or a first-time apply for a brand-new service.

▶
STEP 05

Rollout

Poll ready replicas → live bar → green, or capture logs & undo.

caught for you arm64↔amd64 mismatch · IfNotPresent needs a unique tag · set image won't run your migrator.
